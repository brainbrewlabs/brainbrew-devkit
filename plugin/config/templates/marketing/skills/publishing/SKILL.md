## When to Use

- Formatting finalized content for a specific platform
- Preparing metadata (tags, categories, featured images) for publishing
- Scheduling posts for optimal timing
- Cross-posting adapted versions to multiple channels
- Setting up email newsletter sends

## When NOT to Use

- Writing or creating content (use `content-creator` or `copywriter` skill)
- Editing or proofreading before publishing (use `editing` skill)
- Optimizing content for search engines (use `seo` skill)
- Analyzing post-publish performance (use `analytics` skill)

## Instructions

### 1. Pre-Publish Checklist

Before publishing to any platform, verify:
- [ ] Content has been edited and approved
- [ ] SEO elements are in place (title, meta description, headings)
- [ ] Images are sized correctly for the target platform
- [ ] Links are functional
- [ ] CTA is present and working
- [ ] Author and date are set

### 2. Format for Platform

| Platform | Key formatting tasks |
|----------|---------------------|
| Blog/CMS | Set slug, categories, tags, featured image, excerpt, SEO meta |
| Email | Subject line, preview text, inline styles, plain-text fallback |
| Twitter/X | Trim to 280 chars, attach media, add 1-3 hashtags |
| LinkedIn | Strong first line (pre-fold), professional tone, no hashtag spam |
| Instagram | Visual asset, caption under 2200 chars, 5-10 hashtags in comment |

### 3. Schedule for Optimal Timing

General guidelines (adjust for your audience):
- **Blog**: Tuesday-Thursday, morning
- **Email**: Tuesday-Wednesday, 10 AM local time
- **Twitter/X**: Weekdays 8-10 AM, 12-1 PM
- **LinkedIn**: Tuesday-Thursday 7-8 AM, 12 PM
- **Instagram**: Monday-Friday 11 AM-1 PM

### 4. Cross-Post Adaptation

When distributing to multiple platforms:
1. Start with the full-length version (blog or email)
2. Create platform-specific adaptations (not copies)
3. Adjust tone, length, and format per platform
4. Stagger posts (do not publish everywhere simultaneously)
5. Link back to the canonical version where appropriate

### 5. Post-Publish Verification

After publishing, confirm:
- [ ] Content renders correctly on the platform
- [ ] Links and CTAs work
- [ ] Images display properly
- [ ] No formatting artifacts
- [ ] Analytics tracking is active

## Output

Deliver a publishing report:
- Platform(s) published to
- URLs of published content
- Scheduled date/time if applicable
- Any issues encountered
